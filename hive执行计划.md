
|                     |                    |                                                                                                                      |                                                                                                                                                                                       |
| ------------------- | ------------------ | -------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **`HASH`**          | Map 阶段             | 在内存中使用哈希表进行快速、高效的部分聚合。                                                                                               | 这是最典型的 **Map 端聚合** 模式，`hive.map.aggr=true`（默认开启）时使用[](https://blog.csdn.net/zmx_messi/article/details/135130657)[](https://cloud.tencent.com.cn/developer/article/1779081?from=15425) |
| **`PARTIAL1`**      | Map 阶段             | 也是Map端聚合，但采用的是一般算法，可视为 `HASH` 模式的一种回退策略。                                                                             | 当哈希表无法有效处理（如内存不足）时，或用于处理 `DISTINCT` 的复杂聚合[](https://blog.csdn.net/panfelix/article/details/107683811)                                                                                 |
| **`PARTIAL2`**      | 介于 Map 和 Reduce 之间 | 作为两阶段聚合的中间环节，用于**合并**来自 `PARTIAL1` 的初步聚合结果。                                                                          | 在 `hive.groupby.skewindata=true` 应对数据倾斜时，负责对打散后的数据进行二次聚合[](https://blog.csdn.net/panfelix/article/details/107683811)                                                                  |
| **`PARTIALS`**      | 一个通用的"Partial"模式   | 行为会根据具体聚合函数`DISTINCT`与否动态调整，自动选择 `PARTIAL1` 或 `PARTIAL2`[](https://blog.csdn.net/panfelix/article/details/107683811) | 由Hive优化器在特定复杂场景下（如组合多个`GROUP BY`）自动选择，对用户透明                                                                                                                                           |
| **`COMPLETE`**      | Reduce 阶段          | **单阶段**聚合，所有聚合工作都在**Reduce**端完成。                                                                                     | 当设置 `hive.map.aggr=false` 手动禁用Map端聚合时使用[](https://cloud.tencent.com.cn/developer/article/1779081?from=15425)                                                                          |
| **`MERGERPARTIAL`** | Reduce 阶段          | **合并**来自多个Map任务的**部分聚合**结果，以产生最终结果。                                                                                  | 这是默认开启Map端聚合（`hive.map.aggr=true`）时，Reduce端执行的标准模式[](https://blog.csdn.net/zmx_messi/article/details/135130657)[](https://cloud.tencent.com.cn/developer/article/1779081?from=15425)  |
| **`FINAL`**         | Reduce 阶段          | 在多阶段聚合的**最后一步**，合并来自不同分支的最终结果。                                                                                       | 在两阶段聚合优化（`hive.groupby.skewindata=true`）的**第二个**MapReduce作业的Reduce阶段                                                                                                                  |
